# S03: Menu bar multi-provider usage

**Goal:** Polish the menu bar usage experience so it no longer reads as a Claude-only product when ChatGPT is configured.
**Demo:** The menu bar popover accurately communicates Claude-only, ChatGPT-only, both-provider, loading, and error states.

## Must-Haves

- Usage popover header, cards, refresh controls, empty states, and error states are provider-aware.
- Partial provider configuration is useful rather than treated as setup incomplete.
- Snapshot or view tests cover representative provider combinations.

## Proof Level

- This slice proves: integration

## Integration Closure

MenuBarPopoverView, UsagePopoverView, UsageCardView, ChatGPTUsageCardView, AppModel, and menu icon behavior agree on setup and usage state.
